Why only 6 digits? The units digit of n² only depends on the units digit of n — so there are only 10 cases to check (0–9). The digits 2, 3, 7, 8 never appear. So if any number ends in those digits, it cannot be a perfect square — no calculation needed.

Consequence: Perfect squares can only have 0, 2, 4, 6… trailing zeros. A number ending in an odd number of zeros (like 10, 1000, 100000) is never a perfect square.
Move the slider to build squares by adding L-shaped gnomons. Each gnomon is an odd number of cells. Consecutive odd numbers stack up to make perfect squares.

The gnomon rule: (n+1)² − n² = 2n+1, which is always odd. So going from one perfect square to the next always requires adding an odd number of unit squares.
